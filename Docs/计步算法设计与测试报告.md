## 1. 算法设计

### 1.1 设计目标

设计一套能够**精准识别真实步行动作、剔除日常行为噪声干扰**的计步算法。具体要求：

| 要求 | 说明 |
|------|------|
| 准确性 | 以 MAE / MAPE 为核心指标，标签为文件名标注的步数 |
| 实时性 | 1 秒为单位滑动处理，适配嵌入式平台 |
| 抗干扰 | 能够区分走路/跑步与切菜、刷牙、挥手等日常动作 |
| 可移植 | C 语言实现，单精度浮点，固定大小静态缓冲区 |

### 1.2 算法流程

```
┌──────────────────────────────────────────────────────┐
│                   IMU 原始数据输入                      │
│           (ax, ay, az, gx, gy, gz) × N               │
└──────────────┬──────────────────┬────────────────────┘
               │                  │
         ┌─────▼─────┐     ┌──────▼──────┐
         │ 加速度模值  │     │ 陀螺仪模值   │
         │ |a|=√(x²+y²+z²)│   │ |g|=√(x²+y²+z²)│
         └─────┬─────┘     └──────┬──────┘
               │                  │
         ┌─────▼─────┐     ┌──────▼──────┐
         │ DC 移除 +   │     │  均值滤波    │
         │ 均值滤波(w=7)│     │  (w=25)     │
         └─────┬─────┘     └──────┬──────┘
               │                  │
         ┌─────▼─────┐            │
         │  峰值检测   │            │
         │ (above mean)│           │
         └─────┬─────┘            │
               │                  │
         ┌─────▼─────┐            │
         │ 合并邻近峰  │            │
         │ (无谷间隔)  │            │
         └─────┬─────┘            │
               │                  │
         ┌─────▼─────┐            │
         │ 区间约束    │            │
         │ (15~100样本)│           │
         └─────┬─────┘            │
               │                  │
               └────┬─────────────┘
                    │
              ┌─────▼──────┐
              │ 活动分类门控  │
              │ · 静止检测    │
              │ · 陀螺主导检测 │
              │ · 微弱运动检测 │
              │ · 峰幅检测    │
              └─────┬──────┘
                    │
              ┌─────▼──────┐
              │ 周期性验证   │
              │ (CV of peak │
              │  intervals) │
              └─────┬──────┘
                    │
              ┌─────▼──────┐
              │  步数输出    │
              └────────────┘
```

### 1.3 核心创新

| 创新点 | 原始算法 | 改进算法 |
|--------|----------|----------|
| **信号源** | X/Y/Z 三轴独立处理 | 加速度模值（姿态无关） |
| **轴融合** | 取三轴峰值计数的中位数 ×2 | 单通道直接计数 |
| **噪声过滤** | 无（所有活动同等对待） | 陀螺仪 4 级活动分类门控 |
| **节律验证** | 无 | 峰值间隔 CV 周期性检验 |
| **阈值设定** | 硬编码（`ACC_GRAVITY/10`） | 自适应 + 多级门控 |
| **数据利用** | 仅加速度 | 加速度 + 陀螺仪 |

### 1.4 参数配置

| 参数 | 数值 | 说明 |
|------|------|------|
| `MEAN_WIN` | 7 | 加速度均值滤波窗口（0.14s @ 50Hz） |
| `GYR_SMOOTH` | 25 | 陀螺仪平滑窗口（0.5s） |
| `MIN_PK_DIST` | 15 | 峰值最小间隔（0.3s，对应 3.3Hz 上限） |
| `MAX_PK_DIST` | 100 | 峰值最大间隔（2.0s，对应 0.5Hz 下限） |
| `GYR_STATIONARY` | 300 | 陀螺仪静止阈值 |
| `GYR_LOW_ACTIVE` | 600 | 低活动阈值 |
| `GYR_HIGH_ACTIVE` | 1200 | 高活动阈值 |
| `GYR_ACC_RATIO` | 3.0 | 陀螺/加速比（手臂摆动但身体不动） |

| `PEAK_AMP_MIN` | 200 | 最低峰值幅度 |

| `CV_HIGH_GYR` | 0.8 | 高陀螺时要求的周期性 CV |

## 2. 算法详解

### 2.1 加速度模值计算

使用三轴加速度的欧几里得范数，消除设备佩戴方向的影响：

```
acc_mag[i] = sqrt(ax[i]² + ay[i]² + az[i]²)
```

**优势**：无论手环/手表如何旋转，步行产生的整体加速度变化幅度是一致的。

### 2.2 信号滤波

采用两级处理：

1. **DC 移除**：减去全信号均值，消除重力分量（~4096 LSB = 1G）
2. **均值滤波**：7 样本对称滑动窗口，平滑高频噪声

```
acc_dc[i] = acc_mag[i] - mean(acc_mag)
acc_filt[i] = moving_average(acc_dc, window=7)
```

选择 7 样本窗口的理由：在 50Hz 采样率下，7 样本 ≈ 0.14s，能有效滤除 7Hz 以上噪声，
同时保留 0.5-3Hz 的步行主频成分。

### 2.3 峰值检测

```
1. 找到所有局部极大值（signal[i] ≥ signal[i-1] 且 signal[i] > signal[i+1]）
2. 仅保留高于均值线的峰值（去除无效振荡）
3. 合并无谷间隔的相邻峰（保留更高的那个）
4. 区间约束过滤：峰间距 ∈ [15, 100] 样本
   → 0.3s~2.0s @ 50Hz，对应步频范围 0.5~3.3 Hz
```

区间约束的含义：如果两个峰值过近（<0.3s），可能是同一动作的多次振荡，保留第一个；
如果过远（>2.0s），可能已停止行走，重置计数。

### 2.4 活动分类门控

使用四级门控，逐级过滤非步行活动：

```
Gate A — 静止检测
  条件：gyr_mean < 300
  含义：陀螺仪均值极低 → 设备静止或极微弱运动
  动作：直接返回 0

Gate B — 陀螺主导检测
  条件：gyr_std > 3 × acc_std
  含义：手臂大幅度摆动但身体加速度变化小 → 挥手等
  动作：直接返回 0

Gate C — 微弱运动检测
  条件：acc_std < 150 且 gyr_mean < 600
  含义：加速度波动小且陀螺仪活动低 → 切菜、刷牙等细微动作
  动作：直接返回 0

Gate D — 峰幅检测
  条件：peak_ampl < 200 且 gyr_mean < 800
  含义：峰值幅度过低 → 不足以构成步行冲击
  动作：直接返回 0
```

### 2.5 周期性验证

步行是一种节律性运动，峰值间隔相对均匀。用变异系数（CV = std/mean）度量：

| 陀螺仪活动水平 | 要求的 CV 上限 | 最少峰值数 |
|---------------|---------------|-----------|
| 低（< 600）   | 0.5（严格） | ≥ 2 |
| 中（600~1200） | 0.7（中等） | ≥ 2 |
| 高（≥ 1200）  | 0.8（宽松） | ≥ 2 |

高陀螺活动时放宽 CV 要求，因为跑步时的步频变化比走路大。

## 3. 测试方案

### 3.1 数据集

| 类别 | 文件数 | 步数范围 | 说明 |
|------|--------|----------|------|
| walk/ | 12 | 20~100 | 走路数据（左/右手腕各6份） |
| run/  | 12 | 20~100 | 跑步数据（左/右手腕各6份） |
| others/ | 56 | 0 | 干扰动作（切菜、刷牙、挥手等） |

数据格式：6 轴 IMU，50Hz 采样率，加速度量程 ±8G（4096 LSB/G），陀螺仪量程 ±2000°/s（16.4 LSB/°/s）。

### 3.2 评价指标

| 指标 | 公式 | 适用 |
|------|------|------|
| MAE (Mean Absolute Error) | `Σ|pred_i - true_i| / n` | 所有文件 |
| MAPE (Mean Absolute Percentage Error) | `Σ(|pred_i - true_i| / true_i × 100) / n` | true_i > 0 的文件 |
| FP Rate (False Positive Rate) | `FP / n_noise` | others 类文件 |

| 项目 | Python | C |
|------|--------|---|
| 解释器/编译器 | Python 3.x, NumPy | gcc 12.2.0 -O2 |
| 浮点精度 | float64 (内部) | float32 |
| 结果一致性 | — | 80/80 文件逐项一致（通过 diff 交叉验证） |

## 5. 复杂度分析

### 时间复杂度

设输入样本数为 N。

| 步骤 | 复杂度 | 说明 |
|------|--------|------|
| 加速度/陀螺仪模值 | O(N) | 逐样本平方和开方 |
| 均值滤波 | O(N·w) | 滑动窗口，w=7 或 25 |
| 峰值检测 | O(N) | 单次遍历 |
| 合并邻近峰 | O(P·V) | P=峰值数, V=谷值数, 最坏 O(N²) |
| 区间约束 | O(P) | 单次遍历 |
| 门控与周期性 | O(P) | 统计计算 |
| **总体** | **O(N·w + P·V)** | 实际远小于 N²（P, V ≪ N） |

典型 30 秒数据（1500 样本）：P ≈ 50，V ≈ 50，运算量极小，远超实时。

### 空间复杂度

| 缓冲区 | 大小 | 说明 |
|--------|------|------|
| acc_mag_buf | 195 KB (float[N]) | 加速度模值 |
| acc_filt_buf | 195 KB | 滤波后加速度 |
| gyr_mag_buf | 195 KB | 陀螺仪模值 |
| gyr_smooth_buf | 195 KB | 平滑后陀螺仪 |
| acc_tmp | 195 KB | 临时工作区 |
| gyr_tmp | 195 KB | 临时工作区 |
| peak_idx | 10 KB (uint16_t[5000]) | 峰值索引 |
| valley_idx | 10 KB | 谷值索引 |
| intervals_buf | 10 KB (int16_t[5000]) | 间隔缓冲 |
| **合计** | **~1.2 MB** | |

> **注意**：以上为测试版全量缓冲区（支持 50000 样本 = 1000 秒）。实际嵌入式部署时，可缩减至滑动窗口模式（仅存储 5~10 秒数据），内存占用可降至 ~30 KB。

## 6. 已知局限与改进方向

### 当前局限

1. **周期性节律活动误判**：部分干扰动作（如快速剁菜、有节律的挥手）在加速度和陀螺仪上都表现出与步行相似的节律模式，导致误检。
2. **极端慢走/快跑**：步频超出 0.5~3.3Hz 范围时会被区间约束过滤。
3. **固定阈值**：部分门控阈值（如 `GYR_ACC_RATIO=3.0`）未针对不同佩戴位置自适应。

### 改进方向

1. **频谱分析**：利用 NMSIS DSP 库的 FFT 提取主频特征，步行主频集中在 1~2.5Hz。
2. **机器学习分类**：参考提供的 BP 神经网络示例，训练步行/非步行二分类器。
3. **自适应标定**：前几秒数据用于自动标定步频范围和阈值。
4. **卡尔曼滤波**：用姿态解算分离重力方向，更精确提取身体垂直加速度。
